# Brief: Telegram Mini App — Каталог услуг психолога

> **Версия:** 1.0 (MVP)  
> **Язык интерфейса:** Русский  
> **Стиль:** Telegram-нативный (tg-theme переменные, MainButton, BackButton)  
> **Целевое устройство:** iOS / Android телефон, открывается из Telegram

---

## Принципы дизайна

- Все цвета — через CSS-переменные `--tg-theme-*`, не хардкодить
- Все кнопки — минимум 44×44px (стандарт Apple HIG / Google Material)
- Один главный экшен на экран — через `MainButton` (синяя кнопка Telegram снизу)
- Максимум визуала: фото, иконки, чипы — минимум абзацев текста
- Анимации лёгкие, без heavy blur/shadow (крашит Android бюджетники)
- `WebApp.expand()` вызывается при инициализации — полноэкранный режим всегда

### Экран 1 — Онбординг-квиз

**Цель:** Понять запрос пользователя, подобрать подходящую услугу. Показывается **один раз**.

**Что видит пользователь:**
- Заголовок вопроса (крупно)
- Набор чипов-вариантов ответа
- Прогресс-бар сверху (шаг 1/3, 2/3, 3/3)
- MainButton серая пока не выбран ответ, синяя после выбора

**Вопросы и варианты:**

**Шаг 1:** Что вас привело?
```
[ Тревога и стресс ]  [ Отношения ]
[ Работа и выгорание ]  [ Самооценка ]
[ Жизненный кризис ]  [ Просто поговорить ]
```

**Шаг 2:** Как вам удобнее работать?
```
[ Онлайн — видео ]  [ Онлайн — голос ]
[ Очно ]  [ Пока не знаю ]
```

**Шаг 3:** Вы уже работали с психологом?
```
[ Да, есть опыт ]  [ Нет, впервые ]
```

**Элементы:**
| Элемент | Размер / стиль |
|---------|---------------|
| Прогресс-бар | 4px, `--tg-theme-button-color`, top |
| Заголовок вопроса | 20px bold, `--tg-theme-text-color` |
| Чип-вариант | min 44px высота, border-radius 20px, padding 12px 20px |
| Чип активный | фон `--tg-theme-button-color`, текст белый |
| Чип неактивный | фон `--tg-theme-secondary-bg-color`, текст `--tg-theme-hint-color` |
| MainButton | «Далее» → на последнем шаге «Показать подходящее» |

**Действия:**
- Тап на чип → выделяет выбор, MainButton становится активной
- MainButton → следующий шаг / переход на Главную

**Переход:** → **Главная** (с предвыбранной рекомендацией)

### Экран 6 — Оплата

**Цель:** Принять оплату нативно, не покидая Telegram.

**Реализация:** Telegram Payments API (invoice через бота) + YooKassa / Tinkoff как провайдер.

**Что видит пользователь:**
- Нативный платёжный лист Telegram поднимается снизу (bottom sheet)
- Отображает: название («Сессия с Еленой Чижик»), сумму, метод оплаты
- Если сохранена карта — предлагает оплатить одним тапом
- После нажатия «Оплатить» — спиннер, затем переход

**Ошибки оплаты:**
```
showPopup({
  title: "Оплата не прошла",
  message: "Попробуйте другую карту или обратитесь в поддержку",
  buttons: [
    { id: "retry", type: "default", text: "Попробовать снова" },
    { id: "support", type: "default", text: "Написать в поддержку" }
  ]
})
```

**Переходы:**
- Успех → **Успешное бронирование**
- Ошибка → остаётся с сообщением + кнопки «Повторить» / «Поддержка»
- Закрытие лиса (свайп вниз) → остаётся на экране **Подтверждения**

## Чего НЕ будет в версии 1.0

Эти функции сознательно исключены из MVP — чтобы запустить быстро и не усложнять.

| Функция | Причина отсутствия | Версия |
|---------|-------------------|--------|
| Несколько специалистов / каталог мастеров | MVP — один специалист, без сложной фильтрации | v2 |
| Перенос записи | Сложная логика + занятость слотов. Пока: отмена + новая запись | v2 |
| Покупка пакетов | Апсейл показывается, но оплата недоступна (showAlert «Скоро») | v2 |
| Видео-визитка специалиста | Требует CDN и видеоплеер — отдельная задача | v2 |
| Полный список отзывов | Достаточно 2–3 на главной и профиле | v2 |
| Лист ожидания | Показываем текст «Написать в чат» | v2 |
| Многоязычность | Только русский | v2+ |
| Реферальная программа | «Поделиться с другом» на экране успеха — просто Telegram share | v2 |
| Программы лояльности / бонусы | Счётчик сессий показываем, но механика — позже | v2 |
| Push-уведомления с расписанием | Базовые напоминания через бота — да; cron-рассылки — нет | v2 |
| Групповые сессии | Отдельная логика мест и расписания | v2 |
| Личный кабинет с настройками | Имя берётся из Telegram — нечего настраивать в v1 | v2 |
| Блог / материалы | SEO не нужен в Telegram | v3 |

## Технический стек (рекомендация)

| Слой | Технология |
|------|-----------|
| Frontend | React + Vite (или Vue 3) |
| Telegram SDK | `@twa-dev/sdk` (TypeScript-обёртка) |
| Стили | CSS Modules + CSS-переменные Telegram |
| Иконки | Lucide или phosphor-icons (lightweight) |
| Анимации | CSS transitions + 1 Lottie для экрана успеха |
| Backend | Node.js (Express / Fastify) или Python (FastAPI) |
| База данных | PostgreSQL (записи, клиенты) |
| Оплата | Telegram Payments API + YooKassa |
| Хостинг | любой VPS с HTTPS (обязательно для Mini App) |
| Бот | node-telegram-bot-api или aiogram (Python) |

## Требования к производительности

- Первая отрисовка: < 1.5 сек на 4G
- JS-бандл: < 200KB gzip (lazy load экранов)
- Изображения: WebP, фото специалиста сжать до < 100KB
- Тестировать на: iPhone SE (375px), Android бюджетник (360px), Desktop Telegram (400px sidebar)
